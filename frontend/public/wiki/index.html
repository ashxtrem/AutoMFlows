Welcome to AutoMFlows Documentation

AutoMFlows is a powerful automation tool that allows you to create complex workflows using a visual node-based interface. This documentation provides comprehensive information about all available nodes, their use cases, and configuration options.

Each node represents a specific action or operation in your automation workflow. You can connect nodes together to create sophisticated automation sequences for web automation, API testing, database operations, and more.

Use the navigation sidebar to explore different node categories, or search for specific nodes using the search box above.

Getting Started View API

Start

The entry point of your workflow

Core

The Start node marks the beginning of your automation workflow. It defines global settings that apply to the entire workflow execution.

Visual Reference

Use Cases

  • Workflow Initialization
    Configure browser settings, recording options, and execution parameters before starting automation.
  • Session Recording
    Enable video recording to capture the entire automation session for debugging and documentation.
  • Screenshot Configuration
    Set up automatic screenshots for all nodes or specific timing (pre/post execution).

Configuration Options

recordSessionboolean

Enable video recording of the automation session

screenshotAllNodesboolean

Take screenshots automatically for all nodes

screenshotTiming'pre' | 'post' | 'both'

When to capture screenshots relative to node execution

slowMonumber

Delay in milliseconds between node executions (useful for debugging)

scrollThenActionboolean

Enable smooth scroll to elements before performing actions

{
  "id": "start_1",
  "type": "start",
  "data": {
    "recordSession": true,
    "slowMo": 100,
    "screenshotAllNodes": false
  }
}

Open Browser

Launch and configure a browser instance

Browser

The Open Browser node initializes a browser instance with customizable settings. This is typically the first node after Start in browser automation workflows.

Visual Reference

Use Cases

  • Browser Initialization
    Launch Chromium, Firefox, or WebKit browsers with specific configurations.
  • Viewport Configuration
    Set custom viewport dimensions or maximize the browser window.
  • Stealth Mode
    Enable stealth mode to avoid detection by anti-bot systems.
  • Script Injection
    Inject JavaScript code into all pages loaded by the browser.

Configuration Options

ParameterTypeDescription
headlessbooleanRun browser in headless mode (no visible window)
browser'chromium' | 'firefox' | 'webkit'Select the browser engine to use
viewportWidth / viewportHeightnumberSet browser viewport dimensions
maxWindowbooleanMaximize the browser window
stealthModebooleanEnable stealth mode to avoid detection
jsScriptstringJavaScript code to inject into all pages
capabilitiesobjectBrowser context options (permissions, geolocation, etc.)
launchOptionsobjectBrowser launch options (args, executablePath, etc.)

Navigation

Navigate to URLs and wait for conditions

Browser

Navigate to a specific URL and optionally wait for various conditions before proceeding.

Visual Reference

Use Cases

  • Page Navigation
    Navigate to web pages and wait for them to load completely.
  • Conditional Waiting
    Wait for selectors, URLs, or JavaScript conditions before proceeding.
  • Retry Logic
    Implement retry strategies for unreliable network conditions.

Configuration Options

  • urlstring

    The URL to navigate to

  • waitUntil'load' | 'domcontentloaded' | 'networkidle' | 'commit'

    When to consider navigation successful

  • waitForSelectorstring

    Wait for a specific selector to appear

  • retryEnabledboolean

    Enable retry logic for failed navigations
